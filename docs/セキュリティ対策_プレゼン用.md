# 帳票定義内容チェック — XMLアップロードのセキュリティ対策

プレゼン・説明用のまとめです。

---

## 1. はじめに

### 目的
- ユーザーがアップロードする **比較用XML** および、サーバーから読み込む **基準XML** について、以下のリスクに対応しました。
  - 巨大ファイルによるブラウザ負荷・メモリ枯渇
  - XXE（XML External Entity）攻撃
  - 表示時の XSS（クロスサイトスクリプティング）
  - 想定外形式のXMLによる解析エラー・不正動作

### 前提
- XML の解析・比較は **ブラウザ内（クライアント）** で実施しています。
- 対策の対象は「ブラウザ・セッションの保護」です。

---

## 2. 実施した4つの対策

### 対策① ファイルサイズ制限（最大 5MB）

| 項目 | 内容 |
|------|------|
| **防ぐこと** | 巨大なXMLによるメモリ使用量の増大、フリーズやクラッシュ |
| **実装** | 比較XMLは選択時に `file.size`、基準XMLは fetch 取得後に `text.length` をチェック。5MB（5×1024×1024 バイト）を超えたら読み込み・比較を行わない。 |
| **ユーザーへの表示** | 比較XML：「❌ 5MBを超えるXMLは読み込めません。」／基準XML：同様のメッセージで読み込みを中止。 |

---

### 対策② XXE対策（DOCTYPE / ENTITY の拒否）

| 項目 | 内容 |
|------|------|
| **防ぐこと** | XXE（XML External Entity）攻撃。外部実体の読み込みやローカルファイル参照による情報漏洩・ DoS のリスクを防止。 |
| **実装** | 解析前に XML 文字列を検査。`<!DOCTYPE` または `<!ENTITY` を含む場合は **解析せず拒否**。`sanitizeXmlForParse()` で判定。 |
| **ユーザーへの表示** | アラート：「セキュリティのため、DOCTYPE/ENTITYを含むXMLは利用できません。」 |

※ Designer から通常出力される XML には DOCTYPE/ENTITY は含まれないため、通常利用では影響しません。

---

### 対策③ 表示時のエスケープ（XSS対策）

| 項目 | 内容 |
|------|------|
| **防ぐこと** | XML に含まれる文字列をそのまま HTML に挿入した際の XSS。例：クラスター名に `<script>` 等が含まれるとスクリプトが実行されるリスク。 |
| **実装** | XML 由来の値を画面に出す際、`escapeHtml()` で `& < > " '` を HTML 実体に変換。比較結果・クラスター詳細・ネットワーク詳細・デバッグ表示など、該当するすべての表示箇所で適用。 |
| **ユーザーへの表示** | 記号は「そのまま文字」として表示（例：`A<B` はタグにならず「A<B」と表示）。クラスター詳細モーダルと比較結果タブに「表示について」の説明を追加し、意図しない記号表示が不具合に見えないように配慮。 |

※ Designer から通常出力された XML では、クラスター名などに `<` `>` `&` 等が含まれることはほぼなく、通常は違和感のない表示です。

---

### 対策④ ルート・必須要素の検証

| 項目 | 内容 |
|------|------|
| **防ぐこと** | 想定外のXML構造による解析エラーや、その先の処理の不正動作。 |
| **実装** | 解析直後に `validateXmlStructure()` で次を確認。ルート要素が `conmas` であること、`sheets` 配下に `sheet` が存在すること。条件を満たさない場合は比較処理を実行しない。 |
| **ユーザーへの表示** | アラートで「（基準XML／比較XML）のルート要素が想定外です。」または「～にシート情報が見つかりません。」等を表示。 |

---

## 3. 検証結果（4種類のサンプルXMLで確認）

次の4ファイルで動作を確認し、いずれも **想定どおり** であることを確認しました。

| # | サンプルファイル | 確認内容 | 結果 |
|---|------------------|----------|------|
| 1 | **sample_invalid_structure.xml** | ルートが `conmas` でない・シートなし | ✅ 「比較XMLのルート要素が想定外です。」で比較拒否。構造検証が有効。 |
| 2 | **sample_valid.xml** | 正常な conmas 形式（1シート・2クラスター） | ✅ エラーなく読み込み・比較可能。クラスター詳細も正常表示。 |
| 3 | **sample_xss_test.xml** | クラスター名・選択肢に `<script>` や `& < > " '` を含む | ✅ これらがすべて文字として表示され、スクリプトは実行されない。XSS対策が有効。 |
| 4 | **sample_xxe_reject.xml** | `<!DOCTYPE>` と `<!ENTITY>` を含む | ✅ 「セキュリティのため、DOCTYPE/ENTITYを含むXMLは利用できません。」で読み込み拒否。XXE対策が有効。 |

※ ファイルサイズ制限は、5MB を超える XML を用意して同様に「読み込めません」となることを確認する運用で検証可能です。

---

## 4. まとめ

| 対策 | 一言で |
|------|--------|
| ① ファイルサイズ制限 | 5MB を超える XML は受け付けない。 |
| ② XXE対策 | DOCTYPE/ENTITY を含む XML は解析せず拒否。 |
| ③ XSS対策 | XML 由来の文字はエスケープして表示し、スクリプト実行を防ぐ。 |
| ④ 構造検証 | ルートが `conmas` かつ `sheets`/`sheet` がある場合のみ比較する。 |

- **通常の Designer 出力 XML** では、これらの対策による「違和感のある表示」や「誤って拒否される」ことは想定していません。
- **故意に XML を書き換えて攻撃や不正な形式を仕込んだ場合** に、上記の制限・拒否・エスケープ表示が働く設計です。

サンプルXMLとテスト手順は `public/xml/samples/README.md` に記載しています。
